2. Prometheus API
Prometheus is installed via kube-prometheus-stack in namespace monitoring.

Likely internal service URL:
http://monitoring-kube-prometheus-prometheus.monitoring.svc.cluster.local:9090

Use Prometheus HTTP API:
GET /api/v1/query?query=<PROMQL>

Useful PromQL queries:

Cluster CPU usage:
sum(rate(container_cpu_usage_seconds_total{container!="",pod!=""}[5m]))

Node CPU usage:
sum by (node) (rate(node_cpu_seconds_total{mode!="idle"}[5m]))

Node memory usage:
(node_memory_MemTotal_bytes - node_memory_MemAvailable_bytes) / node_memory_MemTotal_bytes * 100

Pods by phase:
sum by (phase) (kube_pod_status_phase)

Deployments desired replicas:
sum(kube_deployment_spec_replicas)

Deployments available replicas:
sum(kube_deployment_status_replicas_available)

Node readiness:
kube_node_status_condition{condition="Ready",status="true"}

Pod restarts:
sum by (namespace, pod) (increase(kube_pod_container_status_restarts_total[24h]))

Required backend endpoints:
GET /api/metrics/cluster
GET /api/metrics/nodes
GET /api/metrics/pods
GET /api/metrics/restarts

3. ArgoCD API
ArgoCD is installed in namespace argocd.

Likely internal service:
argocd-server.argocd.svc.cluster.local

Try internal URL:
https://argocd-server.argocd.svc.cluster.local

For first version, do not require ArgoCD API auth unless needed.
Alternative: read ArgoCD Application CRDs directly from Kubernetes API:
Group: argoproj.io
Version: v1alpha1
Resource: applications
Namespace: argocd

Collect:
- application name
- sync status
- health status
- repo URL
- target revision
- destination namespace
- last sync time if available

Required backend endpoint:
GET /api/argocd/apps

Expected current ArgoCD app:
- eve-online-tools

4. Loki API
Loki is installed in namespace logging.

Internal service URL:
http://loki-gateway.logging.svc.cluster.local

Use Loki HTTP API:
GET /loki/api/v1/query_range

Example LogQL:
{namespace="eve"}
{namespace=~"eve|argocd|monitoring|logging"} |= "error"

For public dashboard, do not expose raw logs by default.
Only show sanitized summary:
- recent error count
- namespaces with recent errors
- top pods with errors
- last few sanitized error messages if safe

Required backend endpoint:
GET /api/logs/summary
